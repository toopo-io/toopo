# Security Policy

## Reporting a vulnerability

Please **do not open a public GitHub issue** for security vulnerabilities.

Instead, report it privately via GitHub's **"Report a vulnerability"**
(Private Vulnerability Reporting) on the repository's
[Security tab](https://github.com/toopo-io/toopo/security/advisories/new).

For general (non-security) questions, please use
[GitHub Issues](https://github.com/toopo-io/toopo/issues).

Please include:

- A description of the issue and its impact.
- Reproduction steps or a proof-of-concept.
- Any suggested remediation.

## What to expect

- Acknowledgement within 72 hours.
- A coordinated disclosure timeline (typically 30–90 days) once a fix is
  available.
- Credit in the advisory once published, if you wish.

## Scope

This policy covers the Toopo monorepo (`apps/api`, `apps/web`, shared
`packages/*`). Issues in third-party dependencies should be reported
upstream first; we will track and patch downstream.
